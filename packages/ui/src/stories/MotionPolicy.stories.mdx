# Motion Policy Guide

*Control animation intensity based on user needs and context*

## ðŸŽ¯ Motion Philosophy

Our motion system adapts to different contexts and user preferences. We prioritize:

1. **Performance** - Smooth animations that don't impact usability
2. **Accessibility** - Respect user preferences for reduced motion
3. **Context** - Match animation intensity to the user's task
4. **Delight** - Add personality without distraction

| **Page Transitions** | Fade only | Fade + slide | Fade + slide + scale |
| **Stagger Delays** | None | 100ms | 200ms |
| **Spring Physics** | Disabled | Gentle | Bouncy |

## ðŸ”§ Implementation

### Setting Motion Level

Motion level is controlled via `data-motion` attribute on the body element:

```tsx
// app/layout.tsx
export default function RootLayout({ children, pathname }) {
  // Determine motion level based on route
  const getMotionLevel = () => {
    if (pathname.startsWith('/dashboard')) return 'subtle';
    if (pathname === '/') return 'standard';
    if (pathname.startsWith('/demo')) return 'expressive';
    return 'standard';
  };

  return (
    <html lang="ro">
      <body data-motion={getMotionLevel()}>
        {children}
      </body>
    </html>
  );
}
```

## â™¿ Accessibility

### Prefers Reduced Motion

Always respect the user's system preference:

```css
@media (prefers-reduced-motion: reduce) {
  /* Override all motion levels */
  * {
    animation: none !important;
    transition: none !important;
  }
  
  /* Maintain visual feedback without motion */
  .hover-lift:hover {
    box-shadow: 0 0 0 2px var(--primary-warm);
  }
}
```

### JavaScript Hook

```typescript
// hooks/useMotionPolicy.ts
export function useMotionPolicy() {
  const [motionLevel, setMotionLevel] = useState('standard');
  const [prefersReduced, setPrefersReduced] = useState(false);

  useEffect(() => {
    // Check system preference
    const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)');
    setPrefersReduced(mediaQuery.matches);

    // Get current motion level
    const level = document.body.dataset.motion || 'standard';
    setMotionLevel(level);

    // Listen for changes
    const handleChange = (e: MediaQueryListEvent) => {
      setPrefersReduced(e.matches);
    };

    mediaQuery.addEventListener('change', handleChange);
    return () => mediaQuery.removeEventListener('change', handleChange);
  }, []);

  return {
    motionLevel: prefersReduced ? 'subtle' : motionLevel,
    prefersReduced,
    setMotionLevel: (level: string) => {
      if (!prefersReduced) {
        document.body.dataset.motion = level;
        setMotionLevel(level);
      }
    }
  };
}
```

## ðŸ“± Performance Considerations

### Mobile Optimization

```typescript
// Detect device capabilities
const getOptimalMotionLevel = () => {
  // Check for low-end device indicators
  const isLowEnd = 
    navigator.hardwareConcurrency <= 4 ||
    navigator.deviceMemory <= 4 ||
    navigator.connection?.effectiveType === 'slow-2g' ||
    navigator.connection?.effectiveType === '2g';

  if (isLowEnd) return 'subtle';
  
  // Check for touch device (mobile/tablet)
  const isTouchDevice = 'ontouchstart' in window;
  if (isTouchDevice) return 'standard';
  
  return 'expressive';
};
```

### Animation Budget

Keep animations performant by limiting concurrent effects:

| Resource | Subtle | Standard | Expressive |
|----------|--------|----------|------------|
| **Floating Orbs** | 0 | 1-2 | 2-3 |
| **Concurrent Transitions** | 1 | 2-3 | 3-5 |
| **Blur Effects** | Static | Limited | Full |
| **Particle Effects** | None | On interaction | Ambient |
| **3D Transforms** | None | On hover | Active |

## ðŸš€ Best Practices

### Do's âœ…

- **Match motion to user intent** - Productivity tasks need subtle motion
- **Test on real devices** - Ensure smooth performance on target hardware
- **Provide motion controls** - Let users adjust if needed
- **Use motion to guide attention** - Direct focus to important changes
- **Keep it consistent** - Similar actions should have similar animations

### Don'ts âŒ

- **Don't animate everything** - Be selective and purposeful
- **Don't ignore performance** - Monitor FPS and jank
- **Don't force motion** - Respect reduced motion preferences
- **Don't make users wait** - Keep animations snappy
- **Don't loop endlessly** - Ambient animations should be subtle

## ðŸ“Š Motion Metrics

Track these metrics to ensure good motion UX:

| Metric | Target | How to Measure |
|--------|--------|----------------|
| **Frame Rate** | >30fps mobile, >60fps desktop | Chrome DevTools Performance |
| **Jank** | <5% of frames | Performance Observer API |
| **Time to Interactive** | Not delayed by animations | Lighthouse |
| **Motion Sickness Reports** | 0 | User feedback |
| **Preference Changes** | Track subtle vs expressive | Analytics |
